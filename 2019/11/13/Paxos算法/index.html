Paxos算法

发表于 2019-11-13 | 分类于 区块链技术

1.Paxos算法历史及原理

Paxos算法是一种基于消息传递的分布式一致性算法，通过希腊小岛Paxos通过决议的流程来比喻，以下将对该算法进行详细解释。

Paxos算法允许在宕机故障的异步系统中，不要求可靠的消息传递，可容忍消息丢失、延迟、乱序以及重复。它利用大多数 (Majority) 机制保证了2F+1的容错能力，即2F+1个节点的系统最多允许F个节点同时出现故障。

Paxos算法中涉及的角色包括：
| 角色 | 功能 |
| —- | —- |
| Client | 系统外部角色，请求发起者。大众 |
| Proposer | 接受Client请求，向集群提出建议(propose)；并在冲突发生时，起到冲突调节作用。议员 |
| Acceptor(Voter) | 提议投票和接收者，只有在形成法定人数(Quorum，一般即为多数派)时，提议才会被最终接受。国会 |
| Learner | 提议备份者，backup备份，对集群一致性没什么影响。议员 |
